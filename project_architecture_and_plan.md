# Project Architecture and Implementation Plan: Hybrid Lexicon-RoBERTa with RAG for Social Media Analysis

This document outlines a comprehensive system architecture and a step-by-step implementation plan for your project, "A Hybrid Lexicon-RoBERTa Approach to Sentiment and Psychological Analysis on Social Media," incorporating Retrieval-Augmented Generation (RAG). This guide is designed to be directly actionable by an IDE agent, providing clear instructions for setting up and developing the system.

## 1. Project Overview

The project aims to develop an intelligent AI tool for multi-dimensional sentiment and psychological analysis of social media data. It combines traditional lexicon-based methods with the advanced contextual understanding of a fine-tuned RoBERTa model, augmented by a RAG system. The primary objectives include identifying mental health statuses, assisting individuals through early intervention flags, and providing market intelligence for companies [1]. The final deliverable will be a well-documented hybrid model supported by a demonstrative web application.

### 2.1. Architecture Components and Data Flow

1.  **Social Media Platforms (Twitter/X, Reddit)**: The primary sources of raw textual data. Data will be collected from these platforms using their respective APIs.
2.  **Data Collection (APIs)**: This module is responsible for interfacing with social media APIs to fetch raw posts and comments. It handles authentication, query parameters, and initial data retrieval.
3.  **Raw Social Media Data**: The unprocessed data obtained directly from social media platforms. This data often contains noise, such as URLs, hashtags, and special characters.
4.  **Data Preprocessing**: This crucial step cleans and prepares the raw data. It involves removing noise, tokenization, normalization (e.g., lowercasing, slang handling), and potentially initial annotation or labeling if not already present in the dataset.
5.  **Cleaned & Annotated Data**: The output of the preprocessing stage, consisting of high-quality, structured text data suitable for model training and analysis. This data will also be annotated with sentiment and psychological indicators.
6.  **Lexicon (LIWC, NRC)**: External knowledge bases containing psychological and emotional lexicons. These lexicons provide explicit emotional grounding to the analysis.
7.  **Lexicon Features**: Features extracted from the cleaned text using the LIWC and NRC lexicons. These could include counts of words associated with specific emotions, psychological categories, or sentiment scores.
8.  **Pre-trained RoBERTa Model**: A foundational transformer model (e.g., `roberta-base` or `cardiffnlp/twitter-roberta-base-sentiment`) that has been pre-trained on a massive corpus of text. This model provides a strong starting point for contextual understanding.
9.  **Fine-tuned RoBERTa Model**: The pre-trained RoBERTa model fine-tuned on the project's specific cleaned and annotated social media dataset. This specialization enhances its performance for sentiment and psychological analysis in the target domain.
10. **RoBERTa Embeddings**: Contextualized numerical representations (embeddings) of the input text generated by the fine-tuned RoBERTa model. These embeddings capture the semantic meaning and nuances of the text.
11. **Knowledge Base (Psychological Assessments)**: A specialized database containing psychological assessments, mental health guidelines, and expert-annotated examples. This serves as the external knowledge source for the RAG system.
12. **Retriever**: The component of the RAG system responsible for querying the Knowledge Base and retrieving relevant contextual information based on the input social media post. This typically involves similarity search on embeddings.
13. **Augmenter**: This module takes the original input text and the retrieved context from the Knowledge Base and combines them into an augmented input. This augmented input provides richer information to the downstream model.
14. **Hybrid Model**: The core of the system, combining the lexicon features, RoBERTa embeddings, and the RAG-augmented input. This integration allows the model to leverage explicit emotional cues, deep contextual understanding, and external knowledge.
15. **Classification Layer**: A final neural network layer that takes the combined features from the Hybrid Model and outputs the predictions for sentiment (positive/negative/neutral) and specific psychological states (e.g., depression, anxiety, stress).
16. **Sentiment & Psychological Analysis Results**: The final output of the model, including classifications, confidence scores, and potentially flags for concerning content.
17. **Web Application Interface**: A user-friendly interface that allows users to input social media text, view the analysis results, and interact with the system. It will display sentiment, psychological states, and highlight flagged posts.
18. **User**: The end-user interacting with the web application.

## 4. Instructions for the IDE Agent

Your IDE agent should follow these steps to set up the project and implement the system:

### Step 1: Project Setup and Environment

1.  **Create Project Directory**: Create a new directory named `social_media_mental_health_analysis`.
2.  **Initialize Git**: Initialize a Git repository within this directory.
3.  **Create Virtual Environment**: Create and activate a Python virtual environment.
4.  **Install Dependencies**: Install necessary libraries. Start with `transformers`, `torch` (or `tensorflow`), `scikit-learn`, `pandas`, `numpy`, `praw`, `tweepy`, `nrclex`, and a vector database library (e.g., `faiss-cpu` or `chromadb`). Create a `requirements.txt` file and install from it.
5.  **Create Folder Structure**: Create the directories as outlined in Section 3 (`data/`, `models/`, `notebooks/`, `src/`, `config/`).
6.  **Copy Architecture Diagram**: Place `system_architecture.png` in the `project_root/`.

### Step 2: Data Collection and Preprocessing

1.  **`config/config.ini`**: Create this file to store API credentials (Reddit client ID/secret, Twitter bearer token/consumer keys) and other configuration parameters. **Ensure API keys are handled securely and not committed directly to version control.**
2.  **`src/data_collection.py`**: Implement functions to:
    *   Authenticate with Reddit API using `PRAW`.
    *   Authenticate with Twitter API v2 using `tweepy`.
    *   Fetch social media posts and comments based on keywords or user profiles.
    *   Save raw data into `data/raw/`.
3.  **`src/data_preprocessing.py`**: Implement functions to:
    *   Load raw data from `data/raw/`.
    *   Clean text: remove URLs, hashtags, mentions, special characters, and emojis.
    *   Perform tokenization and normalization (lowercasing, slang handling).
    *   Load and integrate publicly available, annotated mental health datasets (e.g., from Kaggle, as referenced in the `kktoh1105` repository) into `data/processed/`.
    *   Ensure data is labeled for sentiment and psychological states.

### Step 3: Lexicon Feature Extraction

1.  **`data/lexicons/`**: Download and place NRC Emotion Lexicon and, if possible, an open-source alternative to LIWC or a parsed LIWC dictionary here.
2.  **`src/lexicon_feature_extraction.py`**: Implement functions to:
    *   Load the NRC Emotion Lexicon (e.g., using `NRCLex`).
    *   Load and parse LIWC-like dictionaries.
    *   Extract features from preprocessed text, such as emotion scores, sentiment scores, and word category counts.

### Step 4: RoBERTa Model Fine-tuning

1.  **`src/roberta_finetuning.py`**: Implement the fine-tuning process:
    *   Load a pre-trained RoBERTa model and its tokenizer (e.g., `AutoModelForSequenceClassification` from `transformers`). Consider `cardiffnlp/twitter-roberta-base-sentiment` for social media data.
    *   Prepare the `Cleaned & Annotated Data` for RoBERTa input (token IDs, attention masks, labels).
    *   Fine-tune the model on the sentiment and psychological classification tasks.
    *   Save the fine-tuned model weights and tokenizer to `models/roberta_finetuned/`.

### Step 5: RAG System Development

1.  **`models/rag_knowledge_base/`**: Prepare a knowledge base. This could be a collection of text documents (e.g., psychological assessment guidelines, mental health articles, expert-annotated social media examples) that are then embedded and indexed using a vector database (e.g., FAISS, ChromaDB).
2.  **`src/rag_system.py`**: Implement:
    *   **Retriever**: A component that takes an input query (social media post) and retrieves the most relevant documents/information from the `rag_knowledge_base` using similarity search on embeddings.
    *   **Augmenter**: A component that combines the original input text with the retrieved context to create an enriched input for the hybrid model. This could involve simple concatenation or more sophisticated prompting techniques if using a separate LLM for augmentation.

### Step 6: Hybrid Model Integration and Training

1.  **`src/hybrid_model.py`**: This module will orchestrate the integration:
    *   Take preprocessed text as input.
    *   Generate `Lexicon Features` using `lexicon_feature_extraction.py`.
    *   Generate `RoBERTa Embeddings` from the `Fine-tuned RoBERTa Model`.
    *   For RAG, use `rag_system.py` to retrieve and augment the input.
    *   Combine these three sets of features (lexicon, RoBERTa, RAG-augmented context). This might involve concatenating them and passing them through a final dense layer for classification.
    *   Define the `Classification Layer` to output sentiment and psychological state predictions.
    *   Implement the training loop for this hybrid model, ensuring it learns from the combined features.

### Step 7: Model Evaluation

1.  **`src/model_evaluation.py`**: Implement functions to:
    *   Load the trained `Hybrid Model`.
    *   Evaluate its performance on a held-out test set using metrics like accuracy, precision, recall, F1-score, and AUC-ROC.
    *   Specifically analyze performance on sarcasm detection and subtle psychological cues.

### Step 8: Web Application Development

1.  **`src/app.py`**: Develop a simple web application (e.g., using Flask or FastAPI):
    *   Create an endpoint to receive social media text input.
    *   Pass the input through the `Hybrid Model` to get predictions.
    *   Display the `Sentiment & Psychological Analysis Results`.
    *   Implement a feature to flag posts identified as highly concerning for mental health.
    *   Provide a user-friendly interface for interaction.

### Step 9: Documentation and Refinement

1.  **`README.md`**: Update with detailed setup instructions, usage examples, and project overview.
2.  **Notebooks**: Ensure all `notebooks/` are clean, well-commented, and reproducible.
3.  **Code Comments**: Add comprehensive comments to all Python scripts.
